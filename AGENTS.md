# AGENTS Knowledge Base: The Master Synthesis (Definitive v1.1)

## 🔴 1. Критические Операционные Ограничения (Must Follow)

### 🌎 1.1. Синтаксис и Окружение
*   **Platform/OS:** Проект жестко привязан к **Windows PowerShell**.
*   **Разделитель команд:** Абсолютно запрещен оператор `&&`. Используется только точка с запятой `;` для последовательности (`cmd1; cmd2`).
*   **Команды Shell:** Строго запрещено использовать синтаксис CMD. Необходимо использовать родные PowerShell командлеты (`Get-ChildItem -Recurse | Select-Object -ExpandProperty FullName`, `Get-Content`).
*   **Интерпретатор Python:** Всегда использовать полный, полный Anaconda-путь (`& 'C:\tools\Anaconda3\python.exe' script.py`).
*   **Время:** Все расчеты и кросс-сравнения жестко привязаны к **UTC**.

### 💾 1.2. Типологический Контракт данных
*   **Источник Истины:** Единственный источник истины — статические JSON дампы из директории `init_dumps/`.
*   **Дата/Уникальность:** Файлы снимков Арены должны иметь формат имени `arena_YYYY-MM-DDTHH-MM-SS.json` для гарантированной лексикографической сортировки.
*   **Локализация (Обязательно):** Любой числовой показатель (рейтинг, урон, дельта) приходит как JSON **STRING**. Десятичный разделитель — **русская запятая (`,`)**.
    *   **Парсинг:** Все расчеты в коде обязаны следовать шаблону: `float("string_value".replace(",", "."))`

## ✨ 2. Ядро Механики и Алгоритмы (Business Rules)

### 📈 2.1. Механика Сравнения (The Accountant's Contract)
1.  **Поток:** Всегда $T_1$ (Базис, старая точка) $\rightarrow T_2$ (Текущая, новая точка).
2.  **Абсолютный Прогресс (Принцип "Bit-Perfect Parity"):** Для расчета накопленного ресурса или очков использовать формулу:
    $$ \text{Прирост} = (\text{Пик\_дня} - \text{База\_дня}) + \text{Конец\_дня} $$
    Это гарантирует видимость прогресса, даже если произошел системный сброс (еженедельный/суточный).
3.  **Сравнение Арены ($\Delta$):** Для расчета явлений (ранг, рейтинг, победы, проигрыши), сравниваются итоговые значения $T_2$ и $T_1$ и вычисляются дельты: $\Delta R$, $\Delta W$, $\Delta L$.

### 🏰 2.2. Игровая Структура и Уровни
*   **Арена**: Сбор данных о составе отрядов через `GetUsersRawInfos` позволяет получать актуальный рейтинг и состав даже для игроков, не входящих в текущий Топ-50.
*   **Уровни (Raw vs UI):** Сырой уровень (`raw_level`: 0-475) должен быть конвертирован в игровой уровень (UI): $\text{UI Level} = (\text{raw\_level} + 1) // 4 + 1$.
*   **Скачки Мощи:** Критический прирост параметров происходит на 1-м шаге каждого нового уровня ($4n-1$).
