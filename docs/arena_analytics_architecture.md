# Архитектура и устройство системы аналитики Арены (Arena Analytics)

Система аналитики Арены представляет собой автономный и расширяемый конвейер (pipeline), спроектированный для сбора, обработки, сравнения и визуализации игровых показателей Топ-50 игроков Арены в игре **Mech Heroes**.

Все ключевые компоненты системы сгруппированы в директории `arena/`, а управление глобальным жизненным циклом (запуск сбора, синхронизация, генерация отчетов и развертывание) оркестрируется главным скриптом [arena_update.py](file:///g:/Video/%21%D0%9C%D0%B5%D0%B4%D0%B2%D0%B5%D0%B4%D0%B8/Mech%20Heroes/%D0%9A%D0%BB%D0%B0%D0%BD%20%D0%9E%D1%80%D0%BA%D0%B8/accountant_bot/arena_update.py) во внешнем окружении Anaconda.

## 2. Безопасность и Оптимизация

### 2.1. Контроль активной сессии (Session Safety)
Оркестратор `arena_update.py` и входящие в него скрипты (`fetch_arena.py`, `fetch_squads.py`, `fetch_and_store_battles.py`) имеют встроенную защиту от дисконнекта:
- **Проверка соединения**: Скрипты проверяют наличие установленного TCP-соединения с сервером игры (`84.201.164.35`).
- **Умное продолжение**: Если соединение активно, оркестратор **не останавливает** работу. Он продолжает выполнение шагов, а скрипты сбора данных просто пропускают сетевые запросы, если не могут найти локальный дамп для замены. Это позволяет обновлять отчеты и делать деплой даже во время твоей игры, используя уже имеющиеся данные.
- **Принудительный запуск**: Флаг `--force` игнорирует проверку (использовать с осторожностью).

### 2.2. Переиспользование дампов и Ручной режим
Для минимизации нагрузки на API и обеспечения работы "в тени" игры реализованы:
1. **Приоритет локальных данных (Local-First)**: Перед любым сетевым запросом скрипт ищет файл `init_*.json` в папке `init_dumps/`. Если файлу меньше 15-30 минут, он используется как источник данных.
2. **Явное указание источника (`--dump`)**: Скрипты `clan_accountant.py` и `fetch_and_store_battles.py` поддерживают аргумент `--dump "путь_к_файлу.json"`. Это позволяет пересобрать любой отчет из конкретного исторического дампа без обращения к серверу и вне зависимости от времени создания файла.

## 3. Разбор компонентов и скриптов

### 2.1. Сбор свежих данных — `arena/fetch_arena.py`
Скрипт [fetch_arena.py](file:///g:/Video/%21%D0%9C%D0%B5%D0%B4%D0%B2%D0%B5%D0%B4%D0%B8/Mech%20Heroes/%D0%9A%D0%BB%D0%B0%D0%BD%20%D0%9E%D1%80%D0%BA%D0%B8/accountant_bot/arena/fetch_arena.py) отвечает за прямое взаимодействие с API игры:
1. **Аутентификация:** Использует жестко прописанные `USER_ID` и `AUTH_KEY` игрока для авторизации.
2. **Получение данных через `/init`:**
   - Отправляет POST-запрос на эндпоинт `/init?userid=227408`.
   - Извлекает блок `userState.arena.leaderboards.cachedPlayers`, содержащий актуальную информацию о Топ-50 игроках на Арене.
   - Считывает метку времени последнего обновления серверных рейтингов: `lastUpdateTime`.
3. **Отказоустойчивое сохранение:**
   - Преобразует `lastUpdateTime` (например, `17/05/2026_00:25:54.123`) в файлобезопасный формат (`arena_17-05-2026T00-25-54.json`).
   - Сохраняет снимок в директорию `arena/snapshots/`.
4. **Обновление рейтингов на сервере:**
   - Дополнительно пытается отправить команду `RefreshArenaLeaderboards` через `/command` для форсирования обновления таблицы лидеров.
   - В случае ошибки 404 (обычное дело при смене сессии или версий) скрипт выводит предупреждение, но не падает, так как первичные данные из `/init` уже успешно сохранены.

### 2.2. Ретроспективный импорт — `arena/sync_from_init.py`
Скрипт [sync_from_init.py](file:///g:/Video/%21%D0%9C%D0%B5%D0%B4%D0%B2%D0%B5%D0%B4%D0%B8/Mech%20Heroes/%D0%9A%D0%BB%D0%B0%D0%BD%20%D0%9E%D1%80%D0%BA%D0%B8/accountant_bot/arena/sync_from_init.py) позволяет догрузить исторические данные из других игровых сессий:
- Сканирует корневую директорию и папки на наличие файлов `current_init_dump.json`, `init_dumps/*.json` и `Ответ на init от сервера.json`.
- Анализирует структуру файлов (поддерживает как сырые ответы сервера с оберткой `data`, так и чистые дампы).
- Безопасно парсит строковое или словарное представление `userState`.
- Если находит уникальный снимок таблицы лидеров, сохраняет его в общий архив `snapshots/` без перезаписи существующих.

### 2.3. Математическое ядро — `arena/analytics_engine.py`
Модуль [analytics_engine.py](file:///g:/Video/%21%D0%9C%D0%B5%D0%B4%D0%B2%D0%B5%D0%B4%D0%B8/Mech%20Heroes/%D0%9A%D0%BB%D0%B0%D0%BD%20%D0%9E%D1%80%D0%BA%D0%B8/accountant_bot/arena/analytics_engine.py) содержит чистую логику сравнения двух снимков времени:
- **`format_power(power_str)`**: Корректно обрабатывает дробные числа с запятой (стандарт RU) и выводит красивый разделитель тысяч (например, `"105495,098"` превращается в `"105 495"`).
- **`calculate_winrate(wins, loses)`**: Рассчитывает процент побед с округлением до двух знаков после запятой.
- **`compare_snapshots(t1_data, t2_data)`**:
  - Принимает два снимка: $T_1$ (базовая, более старая точка) и $T_2$ (сравниваемая, более новая точка).
  - Сортирует игроков в $T_2$ по убыванию рейтинга для присвоения актуальных рангов (1-50).
  - Для каждого игрока вычисляет дельты:
    $$\Delta R = R_{T_2} - R_{T_1} \quad (\text{Изменение рейтинга})$$
    $$\Delta W = W_{T_2} - W_{T_1} \quad (\text{Изменение побед})$$
    $$\Delta L = L_{T_2} - L_{T_1} \quad (\text{Изменение поражений})$$
    $$\Delta Rank = Rank_{T_1} - Rank_{T_2} \quad (\text{Изменение позиции})$$
  - Обратите внимание: для ранга $\Delta Rank > 0$ означает улучшение позиции (например, переход с 5-го места на 2-е дает $+3$ позиции).
  - Если игрок отсутствовал в базе $T_1$, он помечается флагом `is_new = True` (позволяет подсвечивать новичков в таблице лидеров).

### 2.4. Веб-интерфейс и Генератор — `arena/generate_dashboard.py`
Скрипт [generate_dashboard.py](file:///g:/Video/%21%D0%9C%D0%B5%D0%B4%D0%B2%D0%B5%D0%B4%D0%B8/Mech%20Heroes/%D0%9A%D0%BB%D0%B0%D0%BD%20%D0%9E%D1%80%D0%BA%D0%B8/accountant_bot/arena/generate_dashboard.py) использует элегантный и очень производительный подход:
1. **Чтение шаблона:** Читает файл [template.html](file:///g:/Video/%21%D0%9C%D0%B5%D0%B4%D0%B2%D0%B5%D0%B4%D0%B8/Mech%20Heroes/%D0%9A%D0%BB%D0%B0%D0%BD%20%D0%9E%D1%80%D0%BA%D0%B8/accountant_bot/arena/reports/template.html).
2. **Агрегация данных:** Собирает все найденные снимки в один словарь, где ключом выступает оригинальный `timestamp`.
3. **Прямое внедрение (Injection):** Переводит весь собранный массив снимков в компактную JSON-строку и заменяет плейсхолдер `SNAPSHOTS_DATA` прямо внутри HTML.
4. **Результат:** Генерируется полностью автономный (self-contained) интерактивный файл `dashboard.html` размером ~380 КБ, не требующий внешних баз данных или API для рендеринга!

## 3. Устройство интерактивного отчета (HTML / CSS / JS)

Интерфейс отчета в [template.html](file:///g:/Video/%21%D0%9C%D0%B5%D0%B4%D0%B2%D0%B5%D0%B4%D0%B8/Mech%20Heroes/%D0%9A%D0%BB%D0%B0%D0%BD%20%D0%9E%D1%80%D0%BA%D0%B8/accountant_bot/arena/reports/template.html) выполнен по современным стандартам веб-дизайна:
* **Дизайн:** Темная премиальная тема (базовый фон `#1a1a1a`, карточки и элементы управления `#2d2d2d`, золотые акценты `#ffcc00` для лидирующих позиций).
* **Интерактивные селекторы:** 
  - Два выпадающих списка `T1 (База)` и `T2 (Сравнение)` позволяют пользователю выбрать абсолютно любые две точки времени для анализа динамики.
  - По умолчанию система автоматически ставит $T_2$ на самый свежий снимок, а $T_1$ — на начало текущих суток (если снимков много) или на предыдущий снимок (если данных меньше), обеспечивая фокус на актуальной активности за день.
* **Клиентский рендеринг:**
  - Функция `parseInternalDate(s)` разбирает сложный строковый формат даты игры `DD/MM/YYYY_HH:MM:SS.xxxx` для сортировки и расчетов.
  - При смене точек в селекторах JavaScript мгновенно перестраивает таблицу, вычисляя дельты на лету на стороне браузера.
  - **Визуальные маркеры изменений:**
    - Улучшение ранга подсвечивается зеленым значком `↑N`, падение — красным `↓N`.
    - Игроки, вошедшие в Топ-50 впервые за период, подсвечиваются мягким зеленым фоном всей строки и получают плашку `NEW`.
    - Возле измененных показателей рейтинга, мощи, побед и поражений рендерятся цветные дельты в скобках, например, `(+15)` или `(-2)`.

## 4. Публикация и развертывание — `deploy.py`

Скрипт [deploy.py](file:///g:/Video/%21%D0%9C%D0%B5%D0%B4%D0%B2%D0%B5%D0%B4%D0%B8/Mech%20Heroes/%D0%9A%D0%BB%D0%B0%D0%BD%20%D0%9E%D1%80%D0%BA%D0%B8/accountant_bot/deploy.py) автоматизирует публикацию результатов:
1. Загружает переменные окружения FTP из файла `.env` (`FTP_HOST`, `FTP_USER`, `FTP_PASS`, `FTP_PATH`).
2. Рекурсивно проверяет и при необходимости создает удаленную структуру папок на сервере (`public_html/my/mech/`).
3. Загружает сгенерированный файл `arena/reports/dashboard.html` под именем `arena.html`.
4. Также попутно загружает отчет клановой бухгалтерии `clan_monitor/clan_accountant_report.html` под именем `clan.html`, если он существует.
5. Результат доступен всему клану в реальном времени по адресу: [http://ovalhalla.ru/my/mech/arena.html](http://ovalhalla.ru/my/mech/arena.html).

## 5. Сильные архитектурные решения

1. **Клиентская аналитика (No-Backend):** Перенос расчета дельт и сравнений на клиентскую сторону (JavaScript в браузере) позволил избежать сложной логики на сервере и снизить нагрузку на хостинг. Пользователь может переключать временные точки мгновенно без перезагрузки страниц и обращений к базе.
2. **Мягкие отказы API:** Если сервер игры возвращает ошибку на вызов `/command`, скрипт не прекращает работу, а сохраняет данные из `/init`, что делает сбор надежным даже при частых обновлениях протоколов игры.
3. **Бесшовная Anaconda-интеграция:** Использование жестких ссылок на интерпретатор в планировщике/батнике (`C:\tools\Anaconda3\python.exe`) гарантирует стабильный запуск со всеми установленными зависимостями (`requests`, `dotenv` и др.) без конфликтов с системным реестром Windows.
