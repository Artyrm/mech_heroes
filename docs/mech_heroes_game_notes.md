# Mech Heroes — Находки по устройству игры

## Аутентификация

- При загрузке игры через **Yandex Games** Яндекс передаёт разработчикам (Patternmasters) токен аутентификации.
- Игра генерирует для сессии уникальный **`authKey`** — 32-символьный MD5-хэш.
- Пример: `2B8ADCBE7A00EE8AF838139813C3ABBB`
- Ключ может "протухнуть" (сброс сессии, разлогин, принудительный сброс).
- Где взять свежий: открыть игру → F12 → Network → первый запрос `/init` → вкладка Payload → скопировать `authKey`.

## Ключевые эндпоинты API

| Эндпоинт | Описание |
|---|---|
| `/init` | Главный запрос при старте игры. Возвращает полный игровой стейт (~3.7 МБ форматированный JSON). |
| `/command` | Все игровые действия (покупки, бои, апгрейды и т.д.) |

## Структура ответа `/init`

**Верхний уровень:**
```
{
  requestId: int          # ID запроса (пример: 316747767)
  data: {
    sessionID: str        # Пример: "227408_ECB88EB824D73CC58C7F28D2F8E10529"
    time: int             # Серверное время (unix-подобный timestamp)
    commands: list        # Обычно пустой список
    clanData: dict        # 2 ключа: clanState, clanGameVersion
    userState: dict       # 59 ключей — весь игровой стейт игрока
  }
}
```

### `userState` — ключевые секции (59 ключей)
| Ключ | Тип | Описание |
|---|---|---|
| `arena` | dict (10) | Арена: история боёв, рейтинг, дивизион, matchmaking |
| `units` | dict (25) | Все юниты игрока (25 штук) |
| `resources` | dict (23) | Ресурсы (золото, кристаллы и т.д.) |
| `generals` | dict (9) | Генералы |
| `squads` | dict (2) | Отряды |
| `profileState` | dict (6) | Профиль: ник, аватар и т.д. |
| `clanState` | dict (5) | Состояние в клане |
| `playerStatistics` | dict (6) | Общая статистика |
| `lastCommandId` | int | `215420` — ID последней команды |
| `lastCommandTime` | str | `29/04/2026_09:29:51` |
| `usageTime` | str | `29d12h5m` — суммарное время в игре |
| `CurrentPlatform` | str | `YandexGamesDesktop` |

## Арена (`userState.arena`)

**10 ключей:**
| Ключ | Тип | Описание |
|---|---|---|
| `battlesHistory` | list(20) | **История последних 20 боёв** |
| `victories` | int | Всего побед (пример: `853`) |
| `loses` | int | Всего поражений (пример: `759`) |
| `ratingState` | dict | `{rating, lastSetTime}` — текущий рейтинг |
| `division` | dict | Текущий дивизион: очки, место, список противников, время сброса |
| `previousDivision` | dict | Данные прошлого дивизиона |
| `matchmaking` | dict | Кеш противников, последний бой, текущие оппоненты (3 шт), свой состав |
| `league` | dict | Лига (daily, leagueInfo, season) |
| `leaderboards` | dict | Кеш лидеров и время обновления |
| `lastReceivedReward` | dict | Последняя полученная награда |

### Структура одного боя в `battlesHistory`

```
battle = {
  nick: str,              # ник противника
  clanRole: str,          # "Leader" / "Lieutenant" / ...
  avatar: str,            # обычно пустая строка
  avatarConfiguration: {  # внешность персонажа врага
    head, clothes, top, middle, down, frame
  },
  fightTime: str,         # "28/04/2026_21:25:28.9200"
  opponentRating: str,    # рейтинг врага (СТРОКА, не int!)
  ourRatingDelta: str,    # изменение рейтинга (СТРОКА! "14" или "-6")
                          # > 0 = победа, < 0 = поражение
  statistics: {
    bestUnit: int,                   # слот нашего лучшего юнита
    bestUnitAcievements: list[str],  # достижения нашего лучшего юнита
    player: {                        # НАША СТОРОНА
      bestUnit: int,
      lostUnits: int,                # кол-во погибших наших юнитов
      general: {                     # наш генерал
        defId: str,                  # ID генерала (пр: "eva")
        equipables: { id, level, ... }
      },
      units: {                       # наши юниты по слотам (слоты 1-5)
        "1": { state: {...}, statistics: {...} },
        ...
      }
    },
    enemy: {                         # СТОРОНА ВРАГА
      bestUnit: int,                 # слот лучшего юнита врага
      general: { defId, equipables },
      units: {                       # юниты врага по слотам (слоты 6-10)
        "6": { state: {...}, statistics: {...} },
        ...
      }
    }
  }
}
```

**Структура `state` одного юнита:**
```
state = {
  defId: str,       # тип юнита: "fury", "grot", "raptor", "predator", ...
  level: int,       # уровень прокачки (до 476)
  stars: int,       # звёзды (до 15)
  unitType: str,    # "Ordinary" / ...
  isTrial: bool,
  stateId: int,     # уникальный ID экземпляра юнита
  equipables: {     # снаряжение (6 слотов: tracker, armor, weapon, foot, engine, ammunition)
    "<stateId>": {
      id: str,      # название предмета: "watch_the_target_tracker"
      level: int,   # уровень предмета
      itemType: str,# "Gear"
      sharpening: { # заточки по уровням разблокировки
        "1": "e_damage_sharpening",
        "30": "e_crit_sharpening",
        ...
      }
    }
  }
}
```

**Структура `statistics` одного юнита:**
```
statistics = {
  damageDone: str,        # нанесённый урон (может отсутствовать)
  healthLost: str,        # потеряно HP
  healthHealed: str,      # вылечено HP
  killsCount: int,        # убийства (может отсутствовать)
  achievements: list,     # достижения этого юнита в бою
  mergedStatistics: list  # доп. статистика (для юнитов с трансформацией?)
}
```
> **Важно:** `ourRatingDelta`, `opponentRating`, `damageDone`, `healthLost`, `healthHealed` — все **строки**, нужен `int()`/`float()` перед арифметикой. Дробная часть через запятую (русская локаль): `"105495,098"`.

### Matchmaking
- `opponents` — список из 3 ID игроков, предложенных для следующего боя
- `lastFightOpponent` — ID последнего противника
- `cachedOpponents` — полные профили кандидатов с кланами, составами, рейтингами
- `squad` — текущий состав игрока (генерал Eva в примере)

### Division
- `arenaPoints: 1266` — очки текущего дивизиона
- `pastPlace: 6` — место в прошлом (или текущем?) дивизионе
- `resetTime: 30/04/2026_00:00:00` — ежедневный сброс дивизиона
- `opponents` — список из **19** соперников по дивизиону с полными профилями

## HAR-файлы (перехваченные сессии)

| Файл | Размер | Описание |
|---|---|---|
| `старт игры_полный.har` | ~89 МБ | Полный старт игры |
| `старт игры_фильтр_command.har` | ~530 КБ | Только /command запросы |
| `запрос клана.har` | ~14 МБ | Запросы клановых данных |
| `заход на Арену, просмотр статистики.har` | ~21 МБ | Сессия просмотра арены |
| `Ответ на init от сервера.json` | ~972 КБ | Распакованный ответ /init |
| `Ответ на init от сервера_formatterd.json` | ~3.7 МБ | То же, форматированный |
