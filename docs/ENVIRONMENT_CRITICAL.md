# КРИТИЧЕСКИ ВАЖНО: Окружение разработки и ограничения ОС

> **ВНИМАНИЕ:** Обязательно к прочтению перед выполнением любых команд в терминале. Игнорирование этих правил приведет к ошибкам выполнения и поломке конвейера.

## 1. Среда PowerShell (Windows)

Проект работает строго в **Windows PowerShell**. Это кардинально меняет синтаксис команд по сравнению с Bash/Linux.

### Ограничения синтаксиса:
*   **НЕТ `&&`:** В PowerShell оператор `&&` не является корректным разделителем команд. Использование `cmd1 && cmd2` вызовет ошибку `ParserError`.
    *   **Как правильно:** Используйте точку с запятой `;` для последовательного выполнения: `cmd1; cmd2`.
*   **Команда `grep`:** Стандартная команда `grep` из Unix-систем в PowerShell либо отсутствует, либо работает непредсказуемо.
    *   **Как правильно:** Используйте `Select-String`. Пример поиска текста в файлах: 
      `dir -Recurse | Select-String "искомый_текст"`
*   **Пути:** Всегда используйте кавычки для путей, содержащих пробелы (например, `"!Медведи"`).

## 2. Python и Anaconda
- Python установлен через **Anaconda3** по пути `C:\tools\Anaconda3`.
- **НЕ полагайтесь на PATH:** Команды `python` или `py` могут быть не настроены или указывать на другую версию.
- **Всегда используйте полный путь:** В батниках, скриптах и терминале обращайтесь к интерпретатору явно:
  `& 'C:\tools\Anaconda3\python.exe' script.py`
