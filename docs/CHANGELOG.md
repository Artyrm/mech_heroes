# CHANGELOG

Все заметные изменения в этом проекте будут фиксироваться в данном файле.

---

### v0.3.11 (2026-05-25)
- **Алгоритм Истины (Truth Logic):**
  - Полный переход на расчет прироста на основе фиксации факта сброса очков (`vals[j+1] < vals[j]`) для игроков из списка корректировок.
  - Формула при сбросе: `(Пик_дня - База_дня) + Конец_дня`, что позволяет видеть реальный набитый прогресс за сутки.
  - Понедельники теперь всегда начинаются с 0 для всех игроков, обеспечивая корректный учет прогресса после еженедельного игрового сброса.
- **Синхронизация API:**
  - Внедрен цикл `Init -> Refresh -> Init` с 5-секундной паузой для получения самых свежих данных в обход серверного кэша.
  - Использование случайного хэша в командах для гарантированного исполнения на стороне сервера.
- **Восстановление интерфейса (Master UI):**
  - Полностью восстановлен оригинальный «богатый» интерфейс отчетов (O R D A | Analytics Core).
  - Вернулись строки «Исторический рейтинг» (база 11 199 931) и «Суммарный заработок».
  - Восстановлено цветовое кодирование и шрифты Roboto Mono / Orbitron.
  - Возвращены эмодзи 🔥 для индикации «сгоревших» клановых очков.

### v0.2.5 (2026-05-16)
- **Абсолютный паритет данных (Bit-Perfect Parity):**
  - Полная синхронизация алгоритма накопления очков с графическим скриптом (`gen_utc_graph.py`).
  - Унифицирован пайплайн обработки данных: отчет теперь использует тот же список сырых снэпшотов (`sd`), что и графики, исключая расхождения на границах дней.
  - Исправлена логика интерполяции на стыке недель (удалена ошибочная фильтрация уникальных таймстампов, приводившая к потере 1-2 очков).
  - Математическое соответствие: суточный прирост теперь рассчитывается как разница целых значений (`int()`) на границах 00:00 UTC, что гарантирует 100% совпадение с визуальными метками на графиках.
- **Защита от вылета сессии:** Добавлена функция `is_user_active()` для предотвращения конфликтов при активной игре.
- **Улучшение UI:** Добавлена метка времени последнего обновления данных в HTML-отчет.

### v0.2.4 (2026-04-29)

- **Исправление:** Исправлено отображение отрицательного прироста рейтинга.
