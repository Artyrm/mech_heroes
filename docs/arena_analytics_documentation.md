# Документация: Аналитика Арены (Arena Analytics)

Система предназначена для автоматического сбора, анализа и визуализации данных Топ-50 игроков Арены в игре Mech Heroes.

## Архитектура и Компоненты

Система состоит из набора Python-скриптов, которые выполняются последовательно через главный оркестратор `arena_update.py`.

### 1. Сбор данных Арены (`arena/fetch_arena.py`)
*   **Цель**: Получение текущего состояния таблицы лидеров Арены.
*   **Как работает**:
    1.  Делает запрос к эндпоинту `/init` игры для получения актуального состояния (включая `cachedPlayers` в блоке `leaderboards`).
    2.  Пытается вызвать команду `RefreshArenaLeaderboards` через `/command` (в текущей версии отключается или игнорируется, если возвращает 404, так как данные из `/init` обычно достаточно свежие).
    3.  **Дедупликация по контенту**: Чтобы не плодить одинаковые файлы, скрипт вычисляет MD5-хэш по ключевым полям игроков (ID, рейтинг, победы, поражения, мощь). Если хэш совпадает с любым из существующих снимков, сохранение пропускается.
    4.  Если данные новые, они сохраняются в `arena/snapshots/arena_YYYY-MM-DDTHH-MM-SS.json`.

### 2. Синхронизация из дампов (`arena/sync_from_init.py`)
*   **Цель**: Восстановление или наполнение базы снимков Арены из "сырых" дампов `/init`, которые сохраняются другими системами (например, клановым ботом).
*   **Как работает**: Сканирует папку `init_dumps/`, извлекает данные Арены и сохраняет их как снимки, если такого контента еще не было.

### 3. Сбор данных по отрядам (`arena/fetch_squads.py`)
*   **Цель**: Получение детального состава отрядов (мехи, уровни, экипировка) и актуального рейтинга для всех известных системе игроков.
*   **Как работает**:
    1.  Собирает список ID игроков из *всех* известных снимков Арены и локальной базы отрядов (не ограничиваясь только текущим Топ-50).
    2.  Делает запрос к `/directcommand` с типом `GetUsersRawInfos`.
    3.  **Важная находка по API**: Метод `GetUsersRawInfos` возвращает актуальный `arenaRating` (в истории сохраняется как поле `power`) и состав отряда даже для тех игроков, которые выпали из текущего Топ-50 лидерборда игры. Это позволяет нам отслеживать реальный рейтинг интересующих нас игроков постоянно!
    4.  Сохраняет историю изменений отряда в `arena/squads/{userID}/history.json`. Изменения фиксируются только если хэш состава отряда изменился (сравнивается только хэш, метки времени обновляются без дублирования точек).

### 4. Генерация отчетов по отрядам (`arena/generate_squad_reports.py`)
*   **Цель**: Создание веб-страниц для просмотра и сравнения отрядов.
*   **Как работает**: Для каждого игрока генерирует файл `arena/reports/squads/{userID}.html`. Внутри файла содержится вся история его отрядов в формате JSON и JS-логика, которая позволяет выбрать любые две точки во времени и увидеть разницу между ними (подсветка изменений уровней, шмота, заточек).

### 5. Генерация главного дашборда (`arena/generate_dashboard.py`)
*   **Цель**: Создание главной страницы аналитики Арены.
*   **Как работает**: Читает шаблон `arena/reports/template.html`, собирает данные из всех уникальных снимков Арены и генерирует итоговый файл `arena/reports/dashboard.html`. 
*   **Интеграция "выпавших" игроков**: Игроки, которые когда-либо были в базе, но не попали в последний снимок Топ-50, динамически добавляются в конец таблицы. Их место скрывается (отображается `—`), а рейтинг берется самый актуальный — из последнего сбора `fetch_squads.py`. Строки с такими игроками слегка затемнены.

### 6. Деплой (`deploy.py`)
*   **Цель**: Загрузка результатов на веб-сервер по FTP.
*   **Как работает**: Подключается к серверу, загружает `dashboard.html` и все сгенерированные файлы отрядов из папки `squads/`. Файлы загружаются без переименования на лету, чтобы избежать путаницы.

## Ключевые особенности и технические детали
*   **Формат дат ISO 8601**: Файлы снимков Арены имеют формат имен `arena_YYYY-MM-DDTHH-MM-SS.json`. Это критически важно, так как стандартная лексикографическая сортировка `sorted()` в Python без этого ломалась бы (например, 29 число любого месяца сортировалось бы после 17 числа следующего). Внутренняя конвертация дат учитывает часовой пояс МСК.
*   **Умная дедупликация**: Система не создает "пустые" точки на графиках и в таблицах, если за прошедший час на сервере игры ничего не изменилось (сравнение по контентному MD5-хэшу).
*   **Слежение за эволюцией билдов**: Вы можете увидеть, за счет чего игрок поднялся в топе — просто кликните на его ник и сравните состав его отряда вчера и сегодня.
*   **Ядро подавления**: Игроки, у которых в инвентаре генерала найден артефакт "Ядро подавления", отмечаются специальной иконкой в списке Арены. Это позволяет быстро оценить угрозу, так как артефакт значительно меняет баланс сил.
